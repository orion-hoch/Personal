────────────────────────────────────────────────────────────────────────────────
  1. 3D SCENE (the main Canvas)
────────────────────────────────────────────────────────────────────────────────

Canvas / Renderer
  File: src/components/WastelandScene.tsx
  The R3F <Canvas> element that hosts the entire 3D scene.
  Settings: DPR [1, 1.25], antialias off, PCFShadowMap, high-performance GPU.

Ground Plane
  File: src/components/Ground.tsx
  A flat mesh on the XZ plane textured with a procedural pixel-art tile map.
  Uses MeshLambertMaterial. Tile resolution: 64px per tile, 25x25 grid = 1600px.
  Texture uses NearestFilter for the crunchy pixel look.

Tile System
  File: src/engine/tiles.ts
  Procedural tile generator producing 128x64 isometric pixel-art tiles.
  Tile types: ground, ground_dead_grass, ground_scorched, road_ns, road_ew,
    road_cross, rubble, rubble2, puddle, void_edge, void_edge_crumble.
  Color palette defined in the PAL object (muted earth tones).
  Tiles are cached in a Map after first generation.

Grid System
  File: src/engine/gridUtils.ts
  Converts 2D grid coordinates to 3D world positions.
  Constants: TILE_SPACING = 2, GROUND_SIZE = MAP_SIZE * TILE_SPACING = 50.
  Functions: gridTo3D(), getBuildingAnchor().

────────────────────────────────────────────────────────────────────────────────
  3. ATMOSPHERE & LIGHTING
────────────────────────────────────────────────────────────────────────────────

  All defined in: src/components/Atmosphere.tsx

Hemisphere Light
  args: sky #C8A882, ground #2A2520, intensity 0.35
  Adds warm sky / cool ground gradient for depth and fill.

Ambient Light
  intensity: 0.45, color: #9999AA
  Provides uniform base illumination.

Directional Light (Key)
  position: [10, 15, 10], intensity: 0.45, color: #F0D890
  Shadow map: 512x512, frozen after first 4 frames (FreezeStaticShadows).

Directional Light (Fill)
  position: [-8, 10, -6], intensity: 0.18, color: #8899BB
  Cool-toned fill from opposite side to add dimension.

Lighthouse Beacon (LighthouseBeacon component)
  4 point lights at the lighthouse center:
  - Main beacon: y=6.8, #F0E2AE, intensity 12, distance 20
  - 3 cross-fill lights: y=4.2, #DCCB9A, intensity 4, distance 11 (spread +-1.8)

Flicker Lights (FlickerLight component)
  Animated point lights at building/street positions.
  Filter: only lights with intensity >= 0.5 from lightSources array.
  Each has a random phase for independent flicker via sin waves.
  matrixAutoUpdate disabled.

Light Sources Data
  File: src/data/mapData.ts (lightSources array)
  13 total light definitions:
  - 2 lighthouse lights (intensity 1.3, 0.72)
  - 6 building lights (camp, power plant, radio tower, bunker ×2, ferris wheel)
  - 5 street lamps (lower intensity 0.3-0.35, filtered out by threshold)

Ash Particles (AshParticles component)
  70 point sprites falling with drift. Respawn at y=12-17 when hitting ground.
  Uses pointsMaterial, size 0.04, color #B0A898, opacity 0.4.

Ember Sparkles
  drei <Sparkles> placed near the 2 brightest light sources.
  count: 3 per source, scale 2.5, speed 0.2.

Fog
  Linear fog: color #000000, near 25, far 55.
  Fades the scene edges to black.

Post-Processing (EffectComposer)
  - Bloom: threshold 0.65, smoothing 0.2, intensity 0.3, mipmapBlur enabled
  - Vignette: offset 0.25, darkness 0.9

FreezeStaticShadows (helper component)
  Disables shadow map auto-update after 4 frames since the scene is static.

────────────────────────────────────────────────────────────────────────────────
  4. CAMERA
────────────────────────────────────────────────────────────────────────────────

Camera Rig
  File: src/components/CameraRig.tsx
  Default position: [20, 18, 20], FOV 45, near 0.1, far 100.
  Uses drei OrbitControls:
  - enablePan: false
  - enableDamping: true (factor 0.05)
  - minDistance: 8, maxDistance: 45
  - polar angle: PI/6 to PI/2.2

Focus Animation
  When a building is clicked, camera lerps to building's focusOffset position.
  Speed: 0.04 (focus in), 0.12 (return to orbit).
  OrbitControls disabled during focus.
  ESC key returns camera to saved orbit state.

────────────────────────────────────────────────────────────────────────────────
  6. INTERIOR PANELS (content overlays)
────────────────────────────────────────────────────────────────────────────────

Interior Panel
  File: src/components/InteriorPanel.tsx + InteriorPanel.css
  Full-screen modal overlay with backdrop blur.
  Opened when a building is clicked. ESC or backdrop click closes it.
  z-index: 100.

Panel Frame
  CSS class: .interior-frame
  Clip-path polygon with angled corners (sci-fi aesthetic).
  Decorative trim (.interior-frame__trim) and metal plates.
  Max width: 1120px, centered.

Panel Header
  CSS class: .portfolio-header
  Contains: kicker text (building type), h2 title, close button.

Panel Body
  CSS class: .portfolio-body
  Single-column layout (.portfolio-main--single).

Tab Layouts
  Defined in: src/data/content.ts (ContentTab interface)
  Three layout types:
  - "about": profile photo with toggle, bio text, skill groups grid
  - "list": header image, portfolio rows with media plates
  - "contacts": contact card grid

Portfolio Row
  CSS class: .portfolio-row
  Grid: 170px media plate + flexible body.
  Contains: title, organization link, dates, description, skill chips.

Skill Chip
  CSS class: .portfolio-chip
  Inline tag showing a skill/technology name.

Media Plate
  CSS class: .media-plate
  Image container with dark background and subtle noise pattern.
  Uses lazy loading and async decoding.

Contact Card
  CSS class: .contact-card
  Label + value pair for contact information.

Photo Toggle (About panel)
  Component: AboutPhotoToggle
  Toggles between headshot and casual photo with a scan-line transition effect.

────────────────────────────────────────────────────────────────────────────────
  7. VISUALIZATION SEQUENCES (3D walkthroughs)
────────────────────────────────────────────────────────────────────────────────

  Files: src/components/visualization/VisualizationSequence.tsx
         src/components/visualization/SceneAsset.tsx
         src/data/visualizationSequences.ts

Visualization Overlay
  CSS class: .visualization-overlay
  Full-screen takeover with its own Canvas instance.
  Separate from the main wasteland Canvas.

Sequences
  Two defined: "research" and "projects"
  Each has: id, title, intro, accent color, and an array of steps.

Steps
  Each step has: id, title, subtitle, description, asset, tags, links, detail.
  Steps are arranged on a track with 18-unit spacing.

Scene Assets
  Component: SceneAsset (src/components/visualization/SceneAsset.tsx)
  Two kinds:
  - "glb": loaded GLB model (reuses models from public/models/)
  - "shape": procedural geometry with preset shapes

  Procedural Shapes:
    terminal, monolith, archive, dish, frame, capsule, stack,
    sensorRig, weatherTower, buoy, scoreboard, mapTable

Step Camera
  Component: StepCamera
  Follows active step position with mouse-parallax.
  Camera lerp speed: 3.8 (moving) or 6 (settled).

HUD Panel
  CSS class: .visualization-hud
  Shows step title, subtitle, description, tags, links.

Navigation
  CSS class: .visualization-nav
  Prev/next buttons. Keyboard: Arrow keys or W/S.
  Hold to auto-repeat (540ms interval).

Detail Popup
  Component: DetailPopup
  CSS class: .visualization-detail
  Modal with image, purpose, goals, results sections.

  11. PERFORMANCE SETTINGS (current optimized values)
────────────────────────────────────────────────────────────────────────────────

Renderer
  DPR: [1, 1.25] (was [1, 1.5])
  Antialias: OFF (was ON) — Bloom provides sufficient edge smoothing
  Shadow Map: PCFShadowMap (was PCFSoftShadowMap, deprecated)
  Power Preference: high-performance

Shadows
  Shadow map size: 512x512
  Auto-update: FROZEN after 4 frames (FreezeStaticShadows component)
  No objects currently cast shadows (all model meshes: castShadow=false)

Materials
  All GLB model materials: MeshLambertMaterial (was MeshStandardMaterial)
  Ground plane: MeshLambertMaterial (was MeshStandardMaterial)
  Placeholder buildings: MeshLambertMaterial (was MeshStandardMaterial)
  Selection ring: MeshBasicMaterial (cheapest, no lighting needed)

Matrix Updates
  matrixAutoUpdate=false on: internal GLB model children (meshes + groups)
  whose transforms are baked from the GLB file.
  NOT set on: root objects with R3F JSX position/rotation/scale props,
  ground plane, lights — R3F needs auto-update to apply declarative transforms.

Particles
  Ash count: 70 (was 120)
  Sparkles sources: 2 (was 3), 3 particles each (was 4)

Lights
  Lighthouse beacon: 4 lights (was 5) — 1 main + 3 cross-fill
  Active flicker lights: intensity threshold >= 0.45 (same as original)
  Flicker base multiplier: 11x (was 9.5x) — brighter overall
  Hemisphere light: added (new) for sky/ground gradient fill
  Ambient light: 0.45 (was 0.32) — significantly brighter
  Key directional: 0.45 (was 0.24) — nearly doubled
  Fill directional: added (new) — cool #8899BB from opposite side
  Bloom threshold: 0.6 (was 0.65) — catches more glow
  Bloom intensity: 0.35 (was 0.3)
  Vignette darkness: 0.85 (was 0.9) — slightly less edge darkening
  Fog range: 28-58 (was 25-55) — pushed out for more visible scene
  Selection ring segments: 24 (was 48)

Post-Processing
  Bloom: mipmapBlur enabled for better performance
  Vignette: unchanged
